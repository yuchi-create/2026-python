## 程式概念說明

想像舞台上有 4 盞不同顏色的燈，每盞燈有自己的「閃爍週期」：

- 燈 A：每 **2** 拍閃一次
- 燈 B：每 **3** 拍閃一次
- 燈 C：每 **4** 拍閃一次
- 燈 D：每 **6** 拍閃一次

問題：**哪一拍所有燈會同時閃？**

答案就是 LCM（最小公倍數）。而 GCD（最大公因數）代表這些週期之間的最大共同節拍單位。

## 功能規格（詳細）

### F1. 燈光元件

每盞燈由以下元素組成，全部畫在 `Canvas` 上：

1. **主圓形**：半徑約 52px，預設顯示暗色，閃爍時切換為亮色
2. **燈號字母**（A/B/C/D）：顯示於圓心，暗色時灰色、亮色時深色
3. **倒數數字**：顯示於字母下方，顯示「距離下次亮燈還有幾拍」
4. **進度環**（arc）：圍繞圓形的弧線，從滿（剛閃過）逐漸縮小，到下次閃爍時重置為滿
5. **間隔標籤**：顯示於圓形下方，如「每 3 拍」

### F2. 節拍邏輯

- 程式有一個整數計數器 `tick`，初始為 0
- 每次「推進」時 `tick += 1`
- 若 `tick % interval == 0`，該燈閃爍一次
- 閃爍效果：立即變亮，經過約 `tick_ms // 3` 毫秒後自動變暗

### F3. 進度環計算

進度環代表「距離下次閃爍的剩餘比例」：

```
remaining  = interval - (tick % interval)
# 特例：tick % interval == 0 時，remaining = interval（剛閃過，重新倒數）

ring_extent = -(360 × remaining / interval)
# 負號代表順時針方向，從頂部（90°）開始
```

### F4. GCD / LCM 顯示

- 每當任一燈的間隔改變，立即重新計算並更新顯示
- GCD：所有間隔的最大公因數（`math.gcd` + `functools.reduce`）
- LCM：所有間隔的最小公倍數（`a * b // gcd(a, b)` 逐一計算）

### F5. 同步偵測

每拍結束後，偵測本拍有哪些燈亮起：

| 情況 | 顯示文字 | 顏色 |
|------|---------|------|
| 全部 4 盞同步 | `✨ 全部同步！= LCM` | 黃色，畫布短暫閃爍 |
| 2–3 盞同步 | `⚡ A + C 同步`（列出燈名）| 淺藍色 |
| 無同步 | 不顯示 | — |

### F6. 控制元件

| 元件 | 說明 |
|------|------|
| `Spinbox` × 4 | 每盞燈的間隔，範圍 1–20，改變時即時重算 GCD/LCM |
| `Scale`（滑桿） | 控制每拍毫秒數，範圍 150–2000ms，預設 700ms |
| `▶ 開始` | 開始自動推進節拍，按下後變灰 |
| `⏭ 一拍` | 手動推進一拍（暫停狀態才可用）|
| `⏸ 暫停` | 暫停自動推進，按鈕改為「▶ 繼續」|
| `↺ 重置` | tick 歸零，所有燈熄滅，進度環重置 |

## 常見問題

**Q：`root.after()` 怎麼用？**  
A：`root.after(毫秒數, 函式名稱)` — 延遲後執行一次函式。若要重複執行，在函式內再呼叫一次自己。

**Q：為什麼不能用 `time.sleep()`？**  
A：`time.sleep()` 會凍結整個視窗事件迴圈，讓 UI 無法回應。`root.after()` 是 tkinter 的正確計時方式。

**Q：Canvas 的 `create_arc` 參數怎麼看？**  
A：`start` 是起始角度（90 = 12 點鐘方向），`extent` 是延伸角度（負值 = 順時針）。

**Q：Spinbox 的值改變時怎麼觸發函式？**  
A：`Spinbox(command=你的函式)` — 每次點擊上下箭頭都會呼叫。
