> 這就是 **6/22 期末考當天會發給你的那張**。
> 只有步驟標題，**沒有指令、沒有 commit message 範例、沒有 PR 該寫什麼**——
> 那些細節要靠 6/3、6/4 兩堂練習課練熟，自己回想得出來。
>
> 本週請刻意「闔上[完整版](./README.md#期末考-sop-檢查表學生版完整版)、只看這張」跑一次，
> 卡在哪一步、想不起哪個指令，就是你考前還要補的地方。

---

## 八個步驟（順序不能反）

```
[ ] 0. Fork + clone 自己的 fork（第一次才要做）
[ ] 1. 從 main 開 feature 分支
[ ] 2. 跟 AI 拆 ≥3 個 test case（含 ≥1 個 edge case） 
[ ] 3. 寫測試 → 確認紅燈 → commit
[ ] 4. 寫實作 → 跑到綠燈 → commit
[ ] 5. push 到自己的 fork
[ ] 6. 開 PR（你的 fork 分支 → 課程 repo 的 main）
[ ] 7. PR 附 AI_LOG.md
```

## 自我檢測：這些你能不看完整版答出來嗎？

考前對自己問一遍，答不出來就回去翻[完整版](./README.md#期末考-sop-檢查表學生版完整版)：

- 第 1 步的**分支命名**長什麼樣？
- 第 3、4 步的 **commit message 前綴**各是什麼？順序錯會怎樣？
- 「紅燈 → 綠燈」為什麼一定要先紅？一開始就綠代表什麼？
- 第 6 步的 PR 描述**至少要含哪三件事**？
- PR 的 **base 要選誰、compare 要選誰**？選錯會怎樣？

> 這五題答得出來 = 流程分穩了。答不出來 = 你只是抄過、沒有練過。
