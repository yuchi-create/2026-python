## 本週新規則：提示詞自己打

從本週起，課程文件**不再提供 AI 提示詞範例**，理由：

1. 期末考評的是**你**跟 AI 協作的能力，不是你貼範本的能力。
2. `AI_LOG.md` 的「我問 AI 什麼」一欄，本週起必須**逐字貼上你實際輸入的提示詞**（不是事後摘要）。
3. 複製課程文件舊範例、或複製同學的提示詞，在 AI_LOG 中看得出來——千篇一律的提示詞配上「我改了什麼：無」，期末考此項 **0 分**。

打提示詞前先想清楚三件事（這不是範本，是思考順序）：

- 你要 AI 產出**什麼**（測試？實作？解釋？）
- 題目有哪些**條件**是 AI 看不到、你必須講的（函式簽名、範圍、例外要求）
- AI 給的東西你要用**什麼標準**驗收（紅燈？範例值？例外案例有沒有？）

---

## TDD 原則不變（評分依據）

- **先 test 再 impl**：`git log --reverse` 必須先看到 `test:` 再看到 `feat:`。順序反了 = 0 流程分。
- **必須紅燈 → 綠燈**：一開始就綠的測試證明不了任何事。
- **必須開 PR**：fork 分支 → 課程 repo 的 `main`，不開 PR 不算完成。
